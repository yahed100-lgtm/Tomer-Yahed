# AI Usage Documentation

AI tools were used during the project as an assistant for explanation, debugging, documentation, and improving the clarity of the project.

## How AI was used

AI was used for the following purposes:

1. Understanding the theoretical background of catastrophic forgetting.
2. Understanding Elastic Weight Consolidation (EWC) and the role of Fisher information.
3. Explaining the logic behind the graphs from the selected paper.
4. Helping organize the project documentation.
5. Helping write clearer explanations for the README and result interpretation.
6. Helping identify possible problems in repository structure and submission clarity.

## How AI was not used

AI was not used as a replacement for understanding the project.

The project owner was responsible for:

- Choosing the paper.
- Understanding the experiments.
- Running and testing the code.
- Interpreting the results.
- Preparing for the project defense.

## Important clarification

The project code should be evaluated according to what it actually runs and produces. The AI assistance was mainly used to improve explanations and documentation, not to fake results.

The results are generated by running the experiment scripts. Fixed random seeds are used for reproducibility, so similar results are expected between runs.
